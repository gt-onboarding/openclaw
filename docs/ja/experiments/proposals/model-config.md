<div id="model-config-exploration">
  # モデル設定（検討）
</div>

本ドキュメントは、将来のモデル設定に向けた**アイデア**をまとめたものです。正式な仕様ではありません。現在の動作については、次を参照してください：

* [モデル](/ja/concepts/models)
* [モデルフェイルオーバー](/ja/concepts/model-failover)
* [OAuth とプロファイル](/ja/concepts/oauth)

<div id="motivation">
  ## 動機
</div>

オペレーターが求めているもの:

* プロバイダーごとの複数の認証プロファイル（個人用と業務用など）。
* 予測可能なフォールバックを備えたシンプルな `/model` の選択。
* テキストモデルと画像対応モデルの明確な分離。

<div id="possible-direction-high-level">
  ## 想定される方向性（ハイレベル）
</div>

* モデル選択はシンプルに保つ：`provider/model` と、必要に応じたエイリアスだけを使う。
* プロバイダーごとに複数の認証プロファイルを持てるようにし、明示的な優先順位を設定する。
* すべてのセッションが一貫してフェイルオーバーできるよう、グローバルなフォールバックリストを使用する。
* 明示的に設定された場合にのみ、画像ルーティングをオーバーライドする。
