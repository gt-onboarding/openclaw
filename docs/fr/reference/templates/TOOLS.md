<div id="toolsmd-local-notes">
  # TOOLS.md - Notes locales
</div>

Les compétences définissent *comment* les outils fonctionnent. Ce fichier est destiné à *vos* éléments spécifiques — tout ce qui est propre à votre configuration.

<div id="what-goes-here">
  ## Ce que vous pouvez mettre ici
</div>

Des éléments comme :

- Noms et emplacements des caméras
- Hôtes SSH et alias  
- Voix préférées pour la synthèse vocale (TTS)
- Noms de haut-parleurs ou de pièces
- Surnoms des appareils
- Tout ce qui est spécifique à votre environnement

<div id="examples">
  ## Exemples
</div>

```markdown
### Cameras
- living-room → Main area, 180° wide angle
- front-door → Entrance, motion-triggered

### SSH
- home-server → 192.168.1.100, user: admin

### TTS
- Preferred voice: "Nova" (warm, slightly British)
- Default speaker: Kitchen HomePod
```


<div id="why-separate">
  ## Pourquoi séparer ?
</div>

Les compétences sont partagées. Votre configuration vous est propre. Les maintenir séparées signifie que vous pouvez mettre à jour les compétences sans perdre vos notes, et partager des compétences sans exposer votre infrastructure.
