---
title: Déployer sur Railway
---

Déployez OpenClaw sur Railway avec un modèle en un clic et terminez la configuration dans votre navigateur.
C’est la méthode la plus simple « sans terminal sur le serveur » : Railway exécute le Gateway pour vous,
et vous configurez tout via l’assistant web `/setup`.

<div id="quick-checklist-new-users">
  ## Liste de contrôle rapide (nouveaux utilisateurs)
</div>

1) Cliquez sur **Deploy on Railway** (ci-dessous).
2) Ajoutez un **Volume** monté sur `/data`.
3) Configurez les **Variables** requises (au minimum `SETUP_PASSWORD`).
4) Activez le **HTTP Proxy** sur le port `8080`.
5) Ouvrez `https://<your-railway-domain>/setup` et terminez l'assistant de configuration.

<div id="one-click-deploy">
  ## Déploiement en un clic
</div>

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">Déployer sur Railway</a>

Après le déploiement, trouvez votre URL publique dans **Railway → votre service → Settings → Domains**.

Railway fera l’une des choses suivantes&nbsp;:

- vous attribuer un domaine généré (souvent `https://<something>.up.railway.app`), ou
- utiliser votre domaine personnalisé si vous en avez rattaché un.

Ensuite, ouvrez&nbsp;:

- `https://<your-railway-domain>/setup` — assistant de configuration (protégé par mot de passe)
- `https://<your-railway-domain>/openclaw` — Control UI

<div id="what-you-get">
  ## Ce que vous obtenez
</div>

- Gateway OpenClaw hébergé + Control UI
- Assistant de configuration Web à l’adresse `/setup` (aucune commande dans le terminal)
- Stockage persistant via un volume Railway (`/data`) afin que la configuration, les identifiants et l’espace de travail survivent aux redéploiements
- Export de sauvegarde à l’adresse `/setup/export` pour migrer hors de Railway plus tard

<div id="required-railway-settings">
  ## Paramètres requis pour Railway
</div>

<div id="public-networking">
  ### Réseau public
</div>

Activez **HTTP Proxy** pour le service.

- Port : `8080`

<div id="volume-required">
  ### Volume (requis)
</div>

Montez un volume à l’emplacement suivant :

- `/data`

<div id="variables">
  ### Variables
</div>

Définissez ces variables dans le service :

- `SETUP_PASSWORD` (obligatoire)
- `PORT=8080` (obligatoire — doit correspondre au port dans « Public Networking »)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (recommandé)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (recommandé)
- `OPENCLAW_GATEWAY_TOKEN` (recommandé ; à traiter comme un secret d’administration)

<div id="setup-flow">
  ## Flux de configuration
</div>

1) Rendez-vous sur `https://<your-railway-domain>/setup` et saisissez votre `SETUP_PASSWORD`.
2) Choisissez un fournisseur de modèle/d’authentification et collez votre clé.
3) (Facultatif) Ajoutez les jetons Telegram/Discord/Slack.
4) Cliquez sur **Run setup**.

Si les messages privés Telegram sont en mode appairage, l’assistant de configuration peut valider le code d’appairage.

<div id="getting-chat-tokens">
  ## Obtention des jetons de chat
</div>

<div id="telegram-bot-token">
  ### Jeton du bot Telegram
</div>

1) Envoyez un message à `@BotFather` sur Telegram
2) Exécutez la commande `/newbot`
3) Copiez le jeton (du type `123456789:AA...`)
4) Collez-le dans `/setup`

<div id="discord-bot-token">
  ### Jeton de bot Discord
</div>

1) Rendez-vous sur https://discord.com/developers/applications
2) **New Application** → choisissez un nom
3) **Bot** → **Add Bot**
4) Activez **MESSAGE CONTENT INTENT** sous Bot → Privileged Gateway Intents (obligatoire, sans quoi le bot plantera au démarrage)
5) Copiez le **Bot Token** et collez-le dans `/setup`
6) Invitez le bot sur votre serveur (OAuth2 URL Generator ; scopes : `bot`, `applications.commands`)

<div id="backups-migration">
  ## Sauvegardes et migration
</div>

Téléchargez une sauvegarde depuis l’adresse suivante :

- `https://<your-railway-domain>/setup/export`

Cela exporte l’état d’OpenClaw et votre espace de travail, afin que vous puissiez migrer vers un autre hôte sans perdre ni la configuration ni la mémoire.
