---
title: Distribuzione su Render
---

Distribuisci OpenClaw su Render usando l&#39;approccio Infrastructure as Code. Il blueprint `render.yaml` incluso definisce in modo dichiarativo l&#39;intero stack — servizio, disco, variabili d&#39;ambiente — così puoi effettuare il deploy con un solo clic e versionare l&#39;infrastruttura insieme al codice.

<div id="prerequisites">
  ## Prerequisiti
</div>

* Un [account Render](https://render.com) (piano gratuito disponibile)
* Una chiave API del tuo [provider di modelli](/it/providers) preferito

<div id="deploy-with-a-render-blueprint">
  ## Effettua il deploy con un Blueprint di Render
</div>

<a href="https://render.com/deploy?repo=https://github.com/openclaw/openclaw" target="_blank" rel="noreferrer">Effettua il deploy su Render</a>

Facendo clic su questo link:

1. Verrà creato un nuovo servizio Render dal Blueprint `render.yaml` nella directory principale di questo repository.
2. Ti verrà chiesto di impostare `SETUP_PASSWORD`.
3. Verrà creata l&#39;immagine Docker ed eseguito il deploy.

Una volta effettuato il deploy, l&#39;URL del tuo servizio avrà la forma `https://<service-name>.onrender.com`.

<div id="understanding-the-blueprint">
  ## Comprendere il Blueprint
</div>

I blueprint di Render sono file YAML che definiscono la tua infrastruttura. Il `render.yaml` in questo
repository configura tutto ciò che serve per eseguire OpenClaw:

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: PORT
        value: "8080"
      - key: SETUP_PASSWORD
        sync: false          # prompts during deploy
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true  # genera automaticamente un token sicuro
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

Funzionalità Blueprint utilizzate:

| Funzionalità          | Scopo                                                      |
| --------------------- | ---------------------------------------------------------- |
| `runtime: docker`     | Effettua la build dal Dockerfile della repository          |
| `healthCheckPath`     | Render monitora `/health` e riavvia le istanze non integre |
| `sync: false`         | Richiede un valore durante il deploy (segreti)             |
| `generateValue: true` | Genera automaticamente un valore crittograficamente sicuro |
| `disk`                | Archiviazione persistente che sopravvive ai redeploy       |

<div id="choosing-a-plan">
  ## Scelta di un piano
</div>

| Piano | Spegnimento automatico | Disco | Ideale per |
|------|-------------------------|-------|-----------|
| Free | Dopo 15 min di inattività | Non disponibile | Test, demo |
| Starter | Mai | 1GB+ | Uso personale, piccoli team |
| Standard+ | Mai | 1GB+ | Produzione, più canali |

Il Blueprint utilizza il piano `starter` come impostazione predefinita. Per usare il piano gratuito, modifica `plan: free` nel `render.yaml` del tuo fork (ma nota: l’assenza di disco persistente implica che la configurazione venga reimpostata a ogni deploy).

<div id="after-deployment">
  ## Dopo il deployment
</div>

<div id="complete-the-setup-wizard">
  ### Completa la procedura guidata di configurazione
</div>

1. Vai a `https://<your-service>.onrender.com/setup`
2. Inserisci la `SETUP_PASSWORD`
3. Seleziona un provider di modelli e incolla la tua API key
4. Facoltativamente, configura i canali di messaggistica (Telegram, Discord, Slack)
5. Fai clic su **Run setup**

<div id="access-the-control-ui">
  ### Accedi alla Control UI
</div>

La dashboard web è disponibile all&#39;indirizzo `https://&lt;your-service&gt;.onrender.com/openclaw`.

<div id="render-dashboard-features">
  ## Funzionalità della dashboard di Render
</div>

<div id="logs">
  ### Log
</div>

Visualizza i log in tempo reale in **Dashboard → your service → Logs**. Filtra per:

* Log di build (creazione dell&#39;immagine Docker)
* Log di deploy (avvio del servizio)
* Log di runtime (output dell&#39;applicazione)

<div id="shell-access">
  ### Accesso alla shell
</div>

Per il debug, apri una sessione della shell tramite **Dashboard → il tuo servizio → Shell**. Il disco persistente è montato su `/data`.

<div id="environment-variables">
  ### Variabili d&#39;ambiente
</div>

Modifica le variabili in **Dashboard → il tuo servizio → Environment**. Ogni modifica avvia automaticamente un nuovo deploy.

<div id="auto-deploy">
  ### Distribuzione automatica
</div>

Se utilizzi il repository originale di OpenClaw, Render non effettuerà la distribuzione automatica di OpenClaw. Per aggiornarlo, esegui una sincronizzazione manuale del Blueprint dalla dashboard.

<div id="custom-domain">
  ## Dominio personalizzato
</div>

1. Vai a **Dashboard → il tuo servizio → Settings → Custom Domains**
2. Aggiungi il tuo dominio
3. Configura il DNS come indicato (imposta un record CNAME a `*.onrender.com`)
4. Render effettua automaticamente il provisioning di un certificato TLS

<div id="scaling">
  ## Scalabilità
</div>

Render supporta la scalabilità orizzontale e verticale:

* **Verticale**: cambia piano per ottenere più CPU/RAM
* **Orizzontale**: aumenta il numero di istanze (piano Standard e superiori)

Per OpenClaw, la scalabilità verticale è solitamente sufficiente. La scalabilità orizzontale richiede sessioni sticky o una gestione esterna dello stato.

<div id="backups-and-migration">
  ## Backup e migrazione
</div>

Esporta configurazione e spazio di lavoro in qualsiasi momento:

```
https://<your-service>.onrender.com/setup/export
```

Scarica un backup portatile che puoi ripristinare su qualsiasi host OpenClaw.

<div id="troubleshooting">
  ## Risoluzione dei problemi
</div>

<div id="service-wont-start">
  ### Il servizio non si avvia
</div>

Controlla i log di deployment nella dashboard di Render. Problemi comuni:

* `SETUP_PASSWORD` mancante — il Blueprint la richiede, ma verifica che sia impostata
* Porta non corrispondente — assicurati che `PORT=8080` corrisponda alla porta esposta nel Dockerfile

<div id="slow-cold-starts-free-tier">
  ### Avvii a freddo lenti (free tier)
</div>

I servizi del piano gratuito vengono sospesi dopo 15 minuti di inattività. La prima richiesta dopo la sospensione richiede alcuni secondi per l&#39;avvio del container. Passa al piano Starter per mantenere le istanze sempre attive.

<div id="data-loss-after-redeploy">
  ### Perdita di dati dopo il redeploy
</div>

Questo si verifica sul piano gratuito (senza disco persistente). Passa a un piano a pagamento oppure
esporta regolarmente la tua configurazione tramite `/setup/export`.

<div id="health-check-failures">
  ### Errori dei controlli di integrità
</div>

Render richiede una risposta con codice 200 da `/health` entro 30 secondi. Se le build vanno a buon fine ma i deploy falliscono, il servizio potrebbe impiegare troppo tempo ad avviarsi. Verifica:

* I log di build per eventuali errori
* Se il contenitore viene eseguito in locale con `docker build && docker run`
