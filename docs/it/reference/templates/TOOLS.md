<div id="toolsmd-local-notes">
  # TOOLS.md - Note locali
</div>

Le abilità definiscono *come* funzionano gli strumenti. Questo file serve per le *tue* specificità — tutto ciò che è unico nella tua configurazione.

<div id="what-goes-here">
  ## Cosa inserire qui
</div>

Ad esempio:

- Nomi e posizioni delle telecamere
- Host e alias SSH  
- Voci TTS preferite
- Nomi di altoparlanti/stanze
- Nomi personalizzati dei dispositivi
- Qualsiasi altra cosa specifica dell'ambiente

<div id="examples">
  ## Esempi
</div>

```markdown
### Cameras
- living-room → Main area, 180° wide angle
- front-door → Entrance, motion-triggered

### SSH
- home-server → 192.168.1.100, user: admin

### TTS
- Preferred voice: "Nova" (warm, slightly British)
- Default speaker: Kitchen HomePod
```


<div id="why-separate">
  ## Perché tenerli separati?
</div>

Le abilità sono condivise. La tua configurazione è solo tua. Tenerli separati significa che puoi aggiornare le abilità senza perdere le tue note e condividere le abilità senza esporre la tua infrastruttura.
